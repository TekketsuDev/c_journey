# Keyboard Input Forwarding with uinput (QMK-based)

## Description

This project implements a **keyboard-only** input forwarding system on Linux using the
**uinput** kernel interface.
Its main objective is to **inject virtual keyboard events** into the Linux input subsystem based on a predefined keyboard layout, specifically one generated by **QMK firmware** (ZSA Voyager keyboard).

The project focuses on:
- Understand Linux input stack at a low level
- Create a virtual keyboard device via `/dev/uinput` with ZSA source code
- Correct use of a custom QMK layout
- Design pipeline between firmware configuration and host input injection

## Instructions

### Requirements

- Linux system with:
  - `uinput` kernel module available
  - Development headers installed (`libc`, `linux-libc-dev` / `linux-api-headers`)
- Compiler:
  - `gcc` or `clang`
- Permissions:
  - Access to `/dev/uinput` (typically via the `input` group or root)

### Compilation

```bash
  make
```
